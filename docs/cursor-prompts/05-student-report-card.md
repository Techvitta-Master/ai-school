# Task 5 — Build the student Report Card (printable + downloadable PDF)

**Model:** Claude Sonnet 4.6
**Expected time:** 90–120 minutes
**Risk:** Medium (new dependency, print CSS)

## Context
Students (and later parents) should be able to open a beautiful, branded report card for any evaluated test, and download it as PDF. This is the "money shot" of the MVP.

## Goal

### 5.1 Install dependencies
```bash
npm install react-to-print
```
(Stick with `react-to-print` — lightweight, works with Vite, uses browser's native print-to-PDF so no server-side rendering needed.)

### 5.2 Create `src/components/student/ReportCard.jsx`
A fully self-contained component accepting props `{ student, test, evaluation }`. It renders:

**Header band** (gradient indigo → purple):
- School logo/wordmark: "Madavi Institute"
- "Student Report Card"
- Generation timestamp

**Student info row**:
- Photo placeholder circle (initials)
- Name, Class-Section, Roll No
- Test title, date

**Score hero**:
- Huge marks number (e.g. `78 / 100`)
- Grade badge (A+/A/B+…)
- Progress arc

**Per-question table**:
- Columns: Q#, Topic, Scored/Max, Remark
- Color-coded rows by scored/max ratio

**Topic RCA panel** (2-col):
- Left: "Areas to improve" — red card with bulleted weak topics + reasons
- Right: "Strengths" — green card with strong topics

**Improvement plan**:
- Numbered list from `evaluation.details.improvementPlan`

**Teacher feedback callout**:
- The `evaluation.feedback` string in a quote-styled card

**Footer**:
- "Generated by Madavi Institute AI Evaluation System"
- A small watermark for print

### 5.3 Print CSS
Add a `@media print` block in the component (or `index.css`):
- Hide sidebar, header, nav on print
- Ensure the report card fills the page
- Force white background
- Hide "Download PDF" button

### 5.4 Download PDF button
```jsx
import { useReactToPrint } from 'react-to-print';

const ref = useRef();
const handlePrint = useReactToPrint({ content: () => ref.current, documentTitle: `ReportCard_${student.rollNo}_${test.title}` });

<button onClick={handlePrint} className="...">Download PDF</button>
<div ref={ref}><ReportCardBody /></div>
```

### 5.5 Wire into student dashboard
In `src/components/student/StudentScores.jsx`:
- Replace the "AI Feedback" card rendering with a "View Report Card" button per test.
- Clicking opens `ReportCard` in a full-screen modal (`Dialog` from Radix).
- Alternatively, create route `/student/report/:testId` — pick whichever feels better.

## Acceptance criteria
- [ ] Student can click any evaluated test and see a pixel-perfect report card.
- [ ] PDF download produces a clean A4-sized report with no sidebar/header artifacts.
- [ ] Report card looks good both on-screen AND on printed PDF.
- [ ] Works for tests with 0, 1, or many weak topics (graceful empty states).

## Visual references
- [Stripe invoice style](https://stripe.com/docs/invoicing) for clean scoring layout
- Avoid cartoonish fonts — stick with system-ui / Inter.
